Reference System


The Copernicus Reference System Software is a scalable and open-source processing chain.


It has been developed in AGILE since 2021 under Copernicus, a programme of the European Union implemented by the European Space Agency.


This Open-Source solution is today able to manage in parallel the image processing of all the Copernicus Sentinel 1, 2 and 3 (S1, S2 and S3) missions relying on Public Cloud scaling capabilities.

The Reference System Service (RS) is a key component of the CSC Ground Segment, because it provides to the community a complete and open source processing and distribution environment software. Integrated with the Sentinel Data Processors and running on a Commercial Public Cloud, it has the capacity to substitute the nominal CSC Production and Distribution services. Provided by ESA, all Sentinel Data Processors are integrated, validated and published in the frame of the Reference System Service.

The Reference System Service is hosted on a secure Public Commercial Cloud and provides 3 operational functions:

  • sampled systematic Production and Distribution Service;
  • ad-hoc Production and Distribution Service;
  • environment to assure the test, integration, validation, benchmarking of Sentinel Data Processors (new processor versions, new user level data, new Sentinel units).


This software is the exact complement of the ESA-driven Sentinel Data Processors. It provides all the components to integrate them in processing chains and to distribute the output products.


Sentinel Data Processor Publication Service

Copernicus Space Component (CSC) Sentinel Data Processors Publication Service is a key component of the CSC Reference System Service, it offers the environment for:

  • News and announcements about the publication of new or updated versions of Sentinel Data Processors, with notifications for the authenticated users;
  • Search of the published Sentinel Data Processors versions (as authenticated user), as well as defining permissions for download of published Sentinel Data Processors files;
  • REST API based on OData protocol for searching and downloading of the published Sentinel Data Processors files.


The Reference System Software is an assembly of Free Open Source Softwares (FOSS) and ad hoc components developed during the project. The main drivers of the development are:

  • Designing a micro-service architecture.
  • Relying to state-of-the-art FOSS.
  • Making the solution scalable to meet the expected level of performance for each mission and processing level.
  • Creating a Cloud Agnostic solution capable of running on either a Public Cloud infrastructure or a Private Cloud.

At the system level, a first repository named 'Reference-System-Software' presents the system and details how to contribute and deploy the system. At the same system level, an 'Issues' repository is available to manage all issues related to RS Software.

The system itself relies on Infrastructure components to host and run the Reference System Software. This part of the system also includes the Security components necessary to protect the system according to the associated security risk analysis.

This platform takes as inputs the Copernicus S1, S2, and S3 data through each mission’s reception stations and the EDRS Ground Segment, as well as auxiliary data from ESA’s Auxiliary Data Gathering System. Based on these inputs, it uses two types of components to generate User Level Data within the expected timeliness:
  • RS-Core
      • Above the infrastructure some common components have been developed to support the processing of any mission. It includes all following features:
      • Ingoing interfaces (ingestion chain).
      • Outgoing interfaces (the products are made available to the external users through oDATA and STAC APIs).
      • Data Catalog.
      • Compression of data.
      • Lifecycle management.
      • Monitoring (necessary to operate the system).
      • Error Management.
      • User Web Client (for external users to see the production on a Map)

  • RS-AddOn
      • The Instrument Processing Facilities (IPF) embeded in RS AddOns are provided by ESA. AddOns are used to describe each specific processing chain flow. They are a Spring Cloud Data Flow chain that embeds two kinds of processors:
      • The Processor Unit: mainly trigger, prepare worker, router, filter.
      • The Data Processor Unit: mainly execution worker.

      The execution worker wraps Copernicus Sentinel Data Processor binaries and their static data. These AddOns are only accessible to users authorized by ESA, and the procedure to build them is available on GitHub. The Instrument Processing Facilities (IPF) embedded in RS AddOns are provided by ESA.
